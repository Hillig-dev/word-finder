DER STRING-PARAMETER DES KONSTRUKTORS BEZEICHNET DABEI DEN NAMEN DER DA- TEI, AUS DER GELESEN WERDEN SOLL. IN DER PRAXIS REICHT ES MEISTENS NICHT AUS, NUR DEN TYPEN DER KLASSE ZU ADAPTIEREN. VIELMEHR SIND HAEUFIG AUCH METHODEN ANZUPASSEN ODER NEUE HINZUZUFUEGEN. DIES AENDERT ABER NICHTS AM PRINZIP: AD- APTERKLASSEN PASSEN DIE SCHNITTSTELLE EXISTIERENDER BAUSTEINE AN, DAMIT DIESE VON KLIENTEN BENUTZT WERDEN KOENNEN, DIE EINE ANDERE SCHNITTSTELLE ERWARTEN.
HIER NOCH EINE ALTERNATIVE REALISIERUNG DER KLASSE DATEILESER, DIE OHNE SUBCLASSING AUSKOMMT UND TROTZDEM DIE FAEHIGKEITEN DER SCHON VORHANDENEN KLASSE FILEREADER VERWENDET:
PUBLIC CLASS DATEILESER2 IMPLEMENTS CHAREINGABESTROM {
    PRIVATE FILEREADER QUELLE;
PUBLIC DATEILESER2(STRING DATEINAME) THROWS IOEXCEPTION { QUELLE = NEW FILEREADER(DATEINAME);
}
    @OVERRIDE
    PUBLIC INT READ() THROWS IOEXCEPTION {
RETURN QUELLE.READ(); }
}
